# タスク実装ガイド

このコマンドは、タスクを実装する際のチェックリストとガイドラインを提供します。

---

## 実装前のチェックリスト

実装を始める前に、以下を確認してください：

### 1. タスクの理解
- [ ] タスクの目的（何を実現するか）を理解している
- [ ] 期待される成果物が明確である
- [ ] 確認方法が明確である

### 2. 依存関係の確認
- [ ] 前提となるタスクが完了している
- [ ] 必要なリソース（API、データベースなど）が利用可能である

### 3. 技術スタックの確認
- [ ] 使用する技術・ライブラリが決まっている
- [ ] 既存のコードベースの構造を理解している

---

## 実装中のガイドライン

### コーディング規約
- **簡潔さ**: 必要な変更のみを行う
- **可読性**: コメントで意図を説明する
- **保守性**: ベストプラクティスに従う

### 実装の進め方
1. 小さく始める（最小構成で動作確認）
2. テストを書く（可能な場合）
3. エラーハンドリングを追加
4. ログ出力を追加（デバッグ用）

### コミットメッセージ
- 何を変更したかを簡潔に記述
- なぜその変更が必要かを説明（必要に応じて）

---

## 実装後のチェックリスト

### 1. 動作確認
- [ ] タスクで定義された「確認方法」で動作を確認
- [ ] エラーが発生しないことを確認
- [ ] ログ出力が適切であることを確認

### 2. コード品質の確認
- [ ] リンターエラーがないことを確認
- [ ] 不要なコメントやデバッグコードを削除
- [ ] コードレビューの観点で自己チェック

### 3. ドキュメント更新の確認

以下のドキュメントの更新が必要か必ず確認してください：

| ドキュメント | 更新が必要なケース | 更新内容の例 |
|--------------|-------------------|-------------|
| **STEERING.md** | • 技術スタックの追加・変更<br>• アーキテクチャの変更<br>• 重要なトレードオフの決定<br>• 方針の転換 | • 新しいライブラリの選定理由<br>• API 設計の変更背景<br>• なぜこの実装方法を選んだか |
| **DEPLOY.md** | • デプロイ手順の変更<br>• 新しい環境変数の追加<br>• インフラ設定の変更 | • 新しいデプロイコマンド<br>• 環境変数の設定方法<br>• 新しい依存サービスの設定 |
| **README.md** | • セットアップ手順の変更<br>• 使い方の追加・変更<br>• 必要な前提条件の追加 | • 新しい依存パッケージのインストール<br>• 新機能の使い方<br>• 環境構築手順の更新 |
| **BACKLOG.md** | • タスクが完了した場合<br>• 新しい課題が見つかった場合 | • 完了した項目のチェック or 削除<br>• 新しい TODO の追加 |

### 4. 次のステップの確認
- [ ] このタスクの成果物が次のタスクで使えるようになっている
- [ ] 新しい課題や改善点があれば BACKLOG.md に記録

---

## AI の責任

タスク実装後、AI は以下を必ず実施します：

1. **動作確認の実施**
   - タスクで定義された確認方法を実行
   - 結果をユーザーに報告

2. **ドキュメント更新の確認**
   - 上記4つのドキュメントを自動的にチェック
   - 更新が必要な場合: 「◯◯ を更新します」と明示してから更新
   - 更新不要な場合: 「ドキュメント更新は不要です（理由: ◯◯）」と明示

3. **完了報告**
   - タスク完了の報告
   - 次のタスクへの準備状況の確認

4. **タスク状況の表示**
   - 会話内の `/tasks` で生成されたタスクリストを確認
   - 完了したタスクと残りのタスクを表形式で表示
   - 次に実行すべきタスクを明示
   - 進捗率（X / Y タスク完了）を報告

## タスク状況の表示

タスク完了報告の後、AI は以下の情報を自動的に表示します：

### 参照元

- **会話内の `/tasks` コマンドで生成されたタスクリスト**を参照
- BACKLOG.md ではなく、この会話で定義されたタスクを追跡

### 表示内容

1. **完了したタスク**
   - 今回完了したタスクをチェックマーク付きで表示
   - タスク番号、タスク名、概要を明記

2. **残りのタスク**
   - 未完了のタスクをリスト表示
   - 次に実行すべきタスクを明示

3. **進捗状況**
   - 完了したタスク数 / 全タスク数
   - 完了率（パーセンテージ）

4. **注意点**
   - ブロッカーや依存関係
   - 優先度が高いタスク

### 表示フォーマット

```markdown
## 📊 タスク状況

### タスク進捗（会話内の `/tasks` で生成されたリストより）

| # | タスク | 状態 | 概要 |
|---|--------|------|------|
| 1 | タスク名1 | ✅ 完了 | 概要1 |
| 2 | タスク名2 | ✅ 完了 | 概要2 |
| 3 | タスク名3 | 🔄 次はこれ | 概要3 |
| 4 | タスク名4 | ⬜ 未着手 | 概要4 |

### 📈 進捗サマリー
- 完了: 2 / 4 タスク (50%)
- 次のタスク: タスク3

### ⚠️ 注意点
- [あれば、ブロッカーや優先事項]
```

---

## 使い方

1. タスク実装を開始する前に `/implement` を実行
2. 「実装前のチェックリスト」を確認
3. 実装を進める
4. 「実装後のチェックリスト」で確認
5. 完了報告
6. **タスク状況の自動表示**（AI が自動実行）
